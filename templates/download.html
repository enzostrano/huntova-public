Features

Built like the agent you'd hire if you could.

Huntova combines web search, multi-dimensional AI scoring, deep qualification, contact enrichment, and personalised email drafting into one local agent loop. No SaaS subscription, no per-lead cost — just your AI key.

⌕

SearXNG-powered web sweep

50–80 AI-generated queries per hunt across Google, Bing, DuckDuckGo. Domain blocklist + de-dup keep the noise out.

⌬

5-dimensional scoring

Each prospect graded on fit, buyability, timing, reachability, decision-making authority. No generic match scores.

⤓

Playwright deep-qualify

High-fit leads get a multi-page crawl: about, team, contact, recent posts. The agent reads pages like a salesperson would.

✉

Personalised email drafts

Subject + body + LinkedIn note, tailored to evidence the agent actually found. Three tones to pick from. Rewrite on demand.

⚙

Agent DNA + learning loop

The agent builds a profile of your ICP from your wizard answers. Every "good fit / bad fit" rating refines the next hunt.

⌂

Local SQLite storage

Your leads, drafts, hunt runs — all stored at ~/.local/share/huntova/db.sqlite. No cloud database, no privacy compromises.

How it works

Three commands. Then start hunting.

Huntova installs as a single CLI. The first run wizard asks for your provider key; huntova serve opens the dashboard in your browser at http://127.0.0.1:5050.

Install

Initialise

huntova init — pick provider, paste API key, set defaults.

Serve

FAQ

Common questions.

Is Huntova really free?

The CLI is open source under AGPL-3.0 and free to use. You pay only your AI provider's pay-as-you-go pricing (Gemini has a free tier, Claude and OpenAI charge per token). Huntova doesn't take a cut.

What runs locally vs in the cloud?

Everything except the AI calls and web search runs on your machine. Your leads, hunt history, settings, and email drafts all live in a local SQLite file at ~/.local/share/huntova/db.sqlite. Web searches go to a public SearXNG instance; AI scoring goes directly to your chosen provider. Huntova has no central server seeing your data.

Where are my API keys stored?

In your OS keychain when available (macOS Keychain, Windows Credential Manager, Linux Secret Service). Falls back to a Fernet-encrypted file at ~/.config/huntova/secrets.enc with a key derived from machine identifiers. Last-ditch fallback is a 0600-permission JSON file with a stern warning.

Which platforms?

macOS, Windows, and Linux. Python 3.11 or newer required. pipx is the recommended installer because it isolates Huntova's dependencies from your global Python.

What CLI commands are available?

Find & score: huntova examples install tech-recruiting grabs a starter recipe (4 bundled: agencies-eu, b2b-saas-hiring, tech-recruiting, ecommerce-shopify). huntova recipe run <name> replays a saved recipe. huntova hunt --explain-scores runs an ad-hoc hunt and shows per-dimension score breakdowns. huntova ls --filter aurora lists saved leads. huntova lead L3 prints one lead's full detail.

Send: huntova outreach send --top 5 --dry-run previews the AI-drafted emails for your highest-fit leads. Re-run without --dry-run to deliver via your SMTP. huntova outreach send --template ./pitch.txt uses a custom template instead.

Share & track: huntova share --top 10 mints a /h/<slug> URL of your best leads. huntova share status <slug> shows view counts. huntova export dumps CSV or JSON.

Plugins: huntova plugins search hubspot finds plugins in the registry. huntova plugins install <name> one-command install. huntova plugins create my-thing scaffolds a new plugin. huntova plugins contribute opens the registry PR flow.

Ops: huntova serve opens the dashboard. huntova doctor diagnoses provider/keychain/SearXNG/Playwright/plugin issues. huntova metrics show prints daily event funnel (admin-only, needs HV_ADMIN_TOKEN). huntova update self-upgrades via pipx. huntova telemetry enable opts in to anonymous usage stats.

How is this different from Apollo / Hunter / Clay / ZoomInfo?

Those tools sell you access to a static prospect database. Huntova runs a fresh agent every time, finding prospects from the open web matching your specific ICP, then scoring and qualifying each one. Three things win: truth (every lead comes with evidence + source URLs), actionability (drafts and outreach paths included), and control (BYOK, local data, reproducible).

Can I contribute?

Yes — issues + pull requests welcome on GitHub. The repo follows a small-commit cadence; if you're shipping something invasive, open an issue first to align on direction.

What about updates?

pipx upgrade huntova for the released path. git pull if you cloned the repo. The CLI checks for updates on first launch each day and notes when a newer version is on PyPI (no auto-update).
